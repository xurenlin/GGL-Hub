# GGL-Hub 项目架构设计文档

## 📋 文档概述

本文档详细描述了 GGL-Hub 企业级全球化智能物流调度平台的系统架构设计，包括技术选型、微服务划分、数据流设计、部署架构等。

## 🎯 设计目标

### 业务目标
1. **全球化支持**：支持跨国物流业务，处理多语言、多货币、多时区
2. **智能化调度**：利用 AI 技术优化物流路径和资源配置
3. **实时性**：提供实时的订单追踪和状态更新
4. **可扩展性**：支持业务快速增长和功能扩展
5. **高可用性**：确保系统 7x24 小时稳定运行

### 技术目标
1. **微服务架构**：实现服务解耦和独立部署
2. **云原生**：支持容器化部署和弹性伸缩
3. **AI 集成**：深度集成 LangChain4j 等 AI 框架
4. **可观测性**：完善的监控、日志和追踪体系
5. **安全性**：多层次的安全防护机制

## 🔧 微服务架构设计

### 1. 服务划分原则
- **单一职责原则**：每个服务专注于一个业务领域
- **独立部署**：服务可独立编译、打包、部署
- **数据自治**：每个服务拥有自己的数据库
- **服务发现**：通过 Nacos 实现服务注册与发现
- **配置中心**：统一配置管理，支持动态刷新

### 2. 微服务列表

| 服务名称 | 端口 | 职责描述 | 技术栈 |
|---------|------|----------|--------|
| ggl-gateway | 8080 | API 网关，统一入口 | Spring Cloud Gateway |
| ggl-service-order | 8081 | 订单管理服务 | Spring Boot + MyBatis Plus |
| ggl-service-geo | 8082 | 地理信息服务 | Spring Boot + 坐标转换库 |
| ggl-service-i18n | 8083 | 国际化服务 | Spring Boot + 翻译API |
| ggl-service-rag | 8084 | 知识检索服务 | Spring Boot + LangChain4j |
| ggl-service-guardrail | 8085 | AI 安全防护服务 | Spring Boot + AI 评测 |
| ggl-ai-gateway | 8086 | AI 编排网关服务 | Spring Boot + LangChain4j |

### 3. 服务间通信
- **同步调用**：使用 OpenFeign 进行 RESTful API 调用
- **异步通信**：使用 Redis Pub/Sub 或消息队列
- **服务降级**：通过 Hystrix 或 Resilience4j 实现熔断降级
- **负载均衡**：通过 Ribbon 或 Spring Cloud LoadBalancer 实现

## 🗄️ 数据架构设计

### 1. 数据库设计原则
- **分库分表**：按业务领域划分数据库
- **读写分离**：主从复制，读写分离
- **数据一致性**：通过分布式事务或最终一致性保证
- **数据备份**：定期备份，支持数据恢复

### 2. 数据库规划

| 数据库名称 | 用途 | 主要表 | 技术选型 |
|-----------|------|--------|----------|
| ggl_order | 订单数据 | order, order_item, payment_record | MySQL 8.0 |
| ggl_user | 用户数据 | user, user_profile, user_address | MySQL 8.0 |
| ggl_geo | 地理数据 | location, route, address | MySQL 8.0 + 空间索引 |
| ggl_i18n | 国际化数据 | translation, template, resource | MySQL 8.0 |
| ggl_knowledge | 知识库数据 | document, vector, embedding | PostgreSQL + pgvector |
| nacos | 配置中心 | config_info, config_info_aggr | MySQL 8.0 |

### 3. 缓存策略
- **一级缓存**：MyBatis 本地缓存
- **二级缓存**：Redis 分布式缓存
- **缓存穿透**：布隆过滤器 + 空值缓存
- **缓存雪崩**：随机过期时间 + 热点数据永不过期
- **缓存击穿**：互斥锁 + 热点数据预热

## 🔐 安全架构设计

### 1. 认证与授权
- **认证方式**：JWT Token 认证
- **授权机制**：RBAC 基于角色的访问控制
- **单点登录**：支持 OAuth2.0 单点登录
- **会话管理**：Redis 存储会话信息

### 2. 数据安全
- **数据传输**：HTTPS + TLS 1.3 加密
- **数据存储**：敏感数据加密存储
- **数据脱敏**：日志和接口返回数据脱敏
- **SQL 注入防护**：MyBatis 参数化查询

### 3. API 安全
- **API 网关**：统一认证和鉴权
- **限流防护**：令牌桶算法限流
- **防重放攻击**：时间戳 + nonce 校验
- **防 CSRF**：CSRF Token 校验

## 📊 监控与运维架构

### 1. 监控体系
- **应用监控**：Spring Boot Actuator + Micrometer
- **链路追踪**：SkyWalking 或 Jaeger
- **日志收集**：ELK Stack (Elasticsearch + Logstash + Kibana)
- **指标收集**：Prometheus + Grafana

### 2. 告警机制
- **告警规则**：基于阈值和异常模式
- **告警渠道**：邮件、短信、钉钉、企业微信
- **告警分级**：P0-P4 分级告警
- **告警收敛**：防止告警风暴

### 3. 运维工具
- **容器编排**：Docker + Docker Compose
- **持续集成**：Jenkins 或 GitLab CI/CD
- **配置管理**：Ansible 或 Terraform
- **服务网格**：Istio（可选）

## 🚀 部署架构

### 1. 部署环境
- **开发环境**：本地 Docker 环境
- **测试环境**：独立测试集群
- **预发环境**：与生产环境一致
- **生产环境**：高可用集群部署

### 2. 部署策略
- **蓝绿部署**：零停机部署
- **金丝雀发布**：渐进式发布
- **滚动更新**：逐步替换实例
- **回滚机制**：快速回滚到上一版本

### 3. 高可用设计
- **多副本部署**：每个服务至少 2 个实例
- **负载均衡**：Nginx 或云负载均衡器
- **故障转移**：自动故障检测和转移
- **数据备份**：跨可用区数据备份

## 🔄 数据流设计

### 1. 订单创建流程
```
1. 客户端 → API Gateway → Order Service
2. Order Service → Geo Service（地址验证）
3. Order Service → I18n Service（面单生成）
4. Order Service → 数据库（持久化）
5. Order Service → 消息队列（异步通知）
```

### 2. 物流追踪流程
```
1. 客户端 → API Gateway → Order Service
2. Order Service → Geo Service（路径规划）
3. Order Service → 第三方物流 API（实时追踪）
4. Order Service → 数据库（状态更新）
5. Order Service → WebSocket（实时推送）
```

### 3. AI 客服流程
```
1. 客户端 → API Gateway → AI Gateway
2. AI Gateway → RAG Service（知识检索）
3. AI Gateway → Guardrail Service（安全审核）
4. AI Gateway → LLM（智能回复）
5. AI Gateway → 客户端（流式响应）
```

## 📈 扩展性设计

- **无状态服务**：支持水平扩展
- **有状态服务**：通过分片或集群扩展
- **数据库扩展**：分库分表 + 读写分离

- **API 版本管理**：支持多版本 API

## 🛡️ 容错与灾备

### 1. 容错机制
- **服务降级**：非核心功能降级
- **熔断机制**：故障服务快速失败
- **限流保护**：防止系统过载
- **超时控制**：合理设置超时时间

### 2. 灾备方案
- **多活部署**：跨地域多活部署
- **数据同步**：实时数据同步
- **故障切换**：自动故障切换
- **数据恢复**：快速数据恢复

## 📝 技术选型说明

### 1. 核心框架选型
- **Spring Boot 3.4.3**：简化 Spring 应用开发
- **Spring Cloud 2023.0.3**：微服务全家桶
- **Spring Cloud Alibaba 2023.0.1.0**：阿里云微服务解决方案
- **LangChain4j 1.0.0-alpha1**：Java AI 集成框架

### 2. 数据库选型
- **MySQL 8.0**：关系型数据库，事务支持完善
- **Redis 7.0**：高性能缓存，支持丰富数据结构
- **PostgreSQL + pgvector**：向量数据库，支持 AI 检索
- **Elasticsearch 8.12.0**：全文搜索和日志分析

### 3. 中间件选型
- **Nacos 2.3.0**：服务注册与配置中心
- **Spring Cloud Gateway**：API 网关
- **Redisson 3.27.2**：Redis 客户端，支持分布式锁
- **MyBatis Plus 3.5.5**：数据访问层框架
